# Bug Audit: inferrouter

**Date:** 2026-02-24
**Status:** In Progress

## Context

After fixing two bugs (ErrAllFailed без контекста, HealthTracker без reset), проведён полный аудит кодовой базы на аналогичные паттерны.

Уже зафиксировано в `bfca707`:
- RouterError.Tried — per-candidate errors при ErrAllFailed
- HealthTracker.Reset() / ResetAccount()

---

## CRITICAL

### 1. stream.go:95 — Close() возвращает не ту ошибку

`RouterStream.Close()` возвращает ошибку закрытия HTTP body, а не ошибку quota Commit/Rollback. Если stream закрылся ok, но Commit в Redis упал — caller получает `nil`.

**Fix:** Возвращать quota/result error если stream body close прошёл ok.

### 2. router.go:170,206,319 — Commit/Rollback ошибки игнорируются

`_ = r.quotaStore.Rollback(...)` и `_ = r.quotaStore.Commit(...)` — при недоступности Redis/Postgres квоты ломаются без единого лога.

**Fix:** Логировать через Meter (добавить OnQuotaError event или использовать существующий OnResult).

---

## HIGH

### 3. candidate.go:46 — Remaining() ошибка делает free=false

```go
remaining, _ := quotaStore.Remaining(ctx, acc.ID)
free := acc.DailyFree > 0 && remaining > 0
```

Транзиентный сбой Redis → remaining=0 → free=false → все free-аккаунты фильтруются → роутинг на paid.

**Fix:** При ошибке Remaining() считать free=true если DailyFree > 0 (fail-open для free tier).

### 4. postgres.go:226 — SetQuota() игнорирует ошибки

`_, _ = s.pool.Exec(...)` — если таблица не существует или Postgres лежит, инициализация квот проваливается молча.

**Fix:** SetQuota() должен возвращать error.

### 5. redis.go:253-264 — SetQuota() игнорирует ошибки

Exists/HSet без проверки ошибок. То же что #4, но для Redis.

**Fix:** SetQuota() должен возвращать error.

---

## MEDIUM

### 6. openaicompat.go:276 — Malformed SSE chunks молча пропускаются

`json.Unmarshal` fail → `continue`. Ответ неполный, caller не знает.

**Fix:** Считать malformed chunks, при превышении порога возвращать ошибку.

### 7. gemini.go:258 — То же для Gemini streaming

### 8. redis.go:225-228 — ParseInt ошибки → silent zero

Corrupted Redis data → dailyLimit=0 → аккаунт заблокирован.

**Fix:** Возвращать ошибку при невалидных данных.

### 9. openaicompat.go:232 — HTTP error body read fail → потеря контекста

`io.ReadAll` ошибка игнорируется, теряется текст ответа API.

**Fix:** Fallback на status code message если body unreadable.

### 10. gemini.go:221 — То же для Gemini

### 11. health.go:51-61 — RLock→Unlock→Lock race

Между отпусканием RLock и взятием Lock другой goroutine может Reset(). `ah` pointer становится orphaned.

**Fix:** Использовать один Lock вместо RLock→Lock upgrade.

### 12. quota/memory.go:17 — Idempotency seen map растёт весь день

Сброс только при daily reset. При 1M req/day = десятки MB.

**Fix:** TTL-based cleanup или периодическая очистка.
